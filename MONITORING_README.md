# Система мониторинга приложения

## Обзор

Ваше приложение включает в себя полную систему мониторинга на основе Prometheus и Grafana. Система собирает и визуализирует ключевые метрики производительности и состояния приложения.

## Структура файлов мониторинга

- `source/app.py` - реализация метрик в коде приложения
- `grafana_dashboard.json` - готовый дашборд для импорта в Grafana
- `GRAFANA_DASHBOARD_README.md` - подробная документация по дашборду
- `GRAFANA_SETUP_INSTRUCTIONS.md` - пошаговые инструкции по настройке
- `PROMETHEUS_METRICS.md` - полное описание всех метрик
- `METRICS_DASHBOARD_SUMMARY.md` - краткий обзор системы мониторинга

## Метрики, которые собираются

### Статистика контента
- Количество альбомов (`album_count`)
- Количество артикулов (`article_count`)
- Количество файлов (`file_count`)

### Ресурсы системы
- Использование дискового пространства (`disk_usage_bytes_*`)
- Размер базы данных (`database_size_bytes`)
- Время работы приложения (`application_uptime_seconds`)

### Производительность
- HTTP-запросы (`http_requests_total`)
- Время выполнения запросов (`http_request_duration_seconds`)
- Активные подключения (`active_connections`)

## Установка и настройка

Следуйте инструкциям в файле `GRAFANA_SETUP_INSTRUCTIONS.md` для полной настройки системы мониторинга.

## Использование

После настройки системы мониторинга вы сможете:
- Отслеживать производительность приложения в реальном времени
- Анализировать использование ресурсов
- Выявлять потенциальные проблемы до того, как они станут критичными
- Настраивать оповещения для критических ситуаций

## Поддержка и развитие

Система мониторинга автоматически обновляет метрики каждые 30 секунд и предоставляет endpoint `/metrics` для сбора данных Prometheus.
